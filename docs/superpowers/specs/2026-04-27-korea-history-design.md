# 한국역사 학습 웹사이트 — 설계 문서

**작성일:** 2026-04-27  
**프로젝트:** Korea History (한국역사 학습 웹사이트)  
**대상:** 어린이부터 노인까지 전 연령층

---

## 1. 프로젝트 개요

한국의 고조선부터 현대까지 전 시대의 역사를 모든 연령이 함께 학습할 수 있는 웹사이트. 백과사전형 정보, 인터랙티브 탐색 기능, 스토리텔링을 결합한 복합형 플랫폼이다.

### 핵심 목표

- 어린이부터 노인까지 **단일 사이트**에서 각자의 수준에 맞게 학습
- 딱딱한 역사 교과서가 아닌, **살아있는 이야기**로 역사를 전달
- 타임라인·지도·인물 도감으로 역사의 **흐름과 맥락**을 직관적으로 파악

---

## 2. 연령 대응 전략

별도의 어린이 모드 분리 없이 **단일 통합 사이트**로 운영한다. 대신 모든 콘텐츠에 **난이도 배지**를 적용해 독자가 직접 수준을 선택한다.

| 레벨 | 대상 | 특징 |
|------|------|------|
| ⭐ 쉬움 | 초등학생 이하 | 짧은 문장, 쉬운 단어, 스토리 형식 |
| ⭐⭐ 보통 | 중고등학생 | 사건의 원인·결과, 시대적 맥락 포함 |
| ⭐⭐⭐ 심화 | 성인·역사 애호가 | 사료 인용, 학술적 해석, 논쟁점 포함 |

페이지 상단의 레벨 선택기로 난이도를 전환하면 해당 깊이의 내용만 표시된다.

## 6. 콘텐츠 구조

### 파일 레이아웃

```
content/
├── eras/                     # 시대별 MDX (파일 하나에 난이도별 섹션 포함)
│   ├── gojoseon.mdx
│   ├── three-kingdoms.mdx
│   └── ...
├── figures/                  # 인물별 MDX (파일 하나에 난이도별 섹션 포함)
│   ├── sejong.mdx
│   └── ...
└── data/
    ├── eras.json             # 시대 목록 + 메타데이터
    ├── figures.json          # 인물 목록 + 메타데이터
    └── events.json           # 주요 사건 (타임라인용)
```

각 MDX 파일 안에서 `<Level difficulty="easy">`, `<Level difficulty="normal">`, `<Level difficulty="advanced">` 커스텀 컴포넌트로 난이도별 콘텐츠를 구분한다. 페이지에서 선택된 난이도에 해당하는 `<Level>` 블록만 렌더링한다. 파일 분리 방식보다 관리가 단순하고, 시대별 콘텐츠를 한 눈에 볼 수 있다.

## 7. 핵심 기능 상세

### 7-1. 홈 — 타임라인 히어로

- 전체 화면 히어로 영역에 시대 슬라이더 배치
- 시대를 선택하면 해당 시대의 대표 이미지·요약·주요 인물이 전환
- 하단에 역사지도·인물도감·타임라인 바로가기 3개 카드

### 7-2. 인터랙티브 타임라인 (`/timeline`)

- 세로 중앙선 기준 좌우 교차 배치로 사건 카드 표시
- 상단 필터: 전체 / 정치 / 문화 / 전쟁 / 과학 / 인물
- 사건 카드 클릭 시 슬라이드 패널로 상세 내용 표시
- 난이도 배지 표시로 복잡도 안내
- 구현: CSS + Intersection Observer 커스텀 (라이브러리 의존 없이 경량 구현)

### 7-3. 역사 지도 (`/map`)

- 한반도 SVG 지도에 시대별 영토 범위를 색상으로 표시
- 상단 시대 선택기로 전환 시 지도 애니메이션
- 지도 위 핀(마커)으로 주요 전투지·수도·유적지 표시
- 마커 클릭 시 팝업으로 사건 요약 표시
- 구현: `react-simple-maps` + `d3-geo`

### 7-4. 인물 도감 (`/figures`)

- 그리드 카드 목록 (인물 이미지·이름·시대·주요 태그)
- 필터: 시대별 / 역할별(왕·장군·학자·예술가 등)
- 검색: 이름·태그 실시간 검색
- 인물 카드 클릭 시 상세 페이지로 이동
- 상세 페이지: 생애 타임라인, 관련 사건·인물 링크, 난이도별 콘텐츠

### 7-5. 시대 상세 페이지 (`/era/[slug]`)

- 상단: 시대명·기간·대표 이미지
- 난이도 선택기 (쉬움·보통·심화) → MDX 콘텐츠 전환
- MDX 내 커스텀 컴포넌트: `<EraMap>`, `<DifficultyBadge>`, `<FigureCard>`
- 하단: 이 시대의 주요 인물, 다음/이전 시대 네비게이션

## 8. 개발 접근 방식

**방식 2: 핵심 기능 골격 먼저 — 콘텐츠 병행**

### 단계별 구현 순서

1. **골격 셋업** — Next.js 프로젝트 초기화, Tailwind 설정, 공통 레이아웃·헤더
2. **데이터 스키마 확정** — `eras.json`, `figures.json`, `events.json` 구조 정의 및 샘플 데이터 작성
3. **홈 페이지** — 타임라인 히어로 컴포넌트, 시대 슬라이더, 바로가기 카드
4. **시대 상세 페이지** — MDX 렌더링, 난이도 선택기, 관련 인물 카드
5. **인물 도감** — 그리드 목록, 필터·검색, 인물 상세 페이지
6. **타임라인 페이지** — 인터랙티브 연표, 필터, 사건 상세 패널
7. **역사 지도** — SVG 지도, 시대별 영토, 마커 팝업
8. **콘텐츠 채우기** — 각 시대·인물 MDX 파일 작성 (전 시대)
9. **반응형 & 접근성** — 모바일 최적화, 키보드 내비게이션, 큰 글씨 모드
10. **배포** — Vercel 연결, 도메인 설정

## 9. 비기능 요구사항

- **반응형:** 모바일·태블릿·데스크톱 완전 지원
- **접근성:** 큰 글씨 지원, 충분한 색상 대비 (WCAG AA), 키보드 탐색 가능
- **성능:** 정적 생성(SSG)으로 초기 로딩 최소화, 이미지 lazy loading
- **SEO:** 시대·인물 페이지 메타태그 완비
- **다국어:** 초기에는 한국어만, 이후 영어 확장 가능한 구조로 설계
